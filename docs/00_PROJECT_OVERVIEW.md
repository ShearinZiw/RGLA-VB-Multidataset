# 项目总览：RGLA-VB Multi-dataset

## 1. 研究问题

研究对象是小样本条件下的跨域刀具磨损与剩余寿命预测。输入为切削过程多传感器信号、历史走刀序列和可用工艺元数据，输出为连续原始磨损量 `VB` 与剩余走刀次数 `RUL-cut`。域偏移来自刀具、机床、材料、工况和传感器配置变化。

任务定义不变：

- 输出连续原始 `VB` 与 `RUL-cut`，两者地位相同，都不是磨损阶段类别；
- 不以 `VB_norm` 作为训练目标；
- `RUL-cut(e)=max(0,e_EOL-e)`；`e_EOL` 必须来自有来源记录的数据集 EOL 规则，不能默认把最后观测点当失效；
- UDA 中目标域 `VB`、`RUL`、EOL 和删失结局在训练和模型选择时完全不可见；
- 没有可验证 EOL 的样本只参与 `VB` 监督和无标签域适应，`RUL` 损失用 mask 关闭；
- Prefix-30 属于少标签 SSDA，必须与 UDA 分表报告；
- 不从多个 seed 中挑最好结果，正式结果报告全部预注册 seed 的均值、标准差和配对差值。

## 2. 改进框架

完整方法称为 **RGLA-VB/RUL 双回归框架**。旧 `DDGAN + hard pseudo-label + PGRU` 保留为 B2 复现基线；主方法改为六个可独立消融的机制：

1. **生命周期局部增强**：只在 source train 内，对同一刀具/工况且生命周期位置相近的样本做连续增强，避免跨阶段制造物理矛盾样本。
2. **多尺度 1D 特征提取**：建模一次走刀内部的冲击、刀齿周期和低频载荷等不同时间尺度。
3. **生命周期条件 TCN**：用因果膨胀卷积建模跨走刀退化历史，并以可观测连续进程变量进行条件调制。
4. **共享退化表示与双回归头**：一个共享隐状态同时输出原始 `VB` 和 `RUL-cut`，分别表达当前磨损与到 EOL 的距离。
5. **RGLA / DARE-GRAM**：在相近生命周期位置对齐与连续回归有关的几何结构，避免全局对齐混合不同磨损水平。
6. **不确定性感知 EMA Teacher**：用参数指数移动平均产生目标域软连续预测，并按 VB/RUL 预测不确定性降低不可靠目标一致性损失。

完整公式、标签 mask 和开关见 `docs/01_METHOD_RGLA_VB.md`。各组件均为已有技术；项目创新主张只建立在生命周期约束、双回归可靠性和回归结构对齐的组合及其实证消融上。

## 3. 证据梯度

| 层级 | 数据 | 作用 | 结论边界 |
|---|---|---|---|
| 开发 | PHM2010 C1/C4/C6 | 完整寿命上的 VB/RUL 双回归与机制消融 | 跨刀具、同数据集 |
| 独立主验证 | Hannover M1/M2/M3 | 6 个真实跨机床任务；EOL来源审计通过后评价双任务 | 跨机床泛化 |
| 低成本压力测试 | NASA Milling | 8 个同工况跨材料任务；RUL按右删失处理 | 小样本 VB 泛化，不承担完整RUL结论 |

NASA 不能替代 Hannover：它只有 16 个 case、167 个 run，21 个 run 缺少 `VB`，且不是可靠的 run-to-failure 数据。它适合做低成本 VB 外部方向性验证和删失鲁棒性检查，不能生成虚构的完整 RUL 标签。

## 4. 目录职责

| 路径 | 职责 |
|---|---|
| `legacy/scripts/` | 已有 PHM2010 复现与导出脚本；默认冻结 |
| `src/rgla_vb/data/` | 数据契约、注册表和数据集适配器 |
| `configs/` | 数据路径和预注册实验配置 |
| `docs/` | 方法、数据、实验和验收文档 |
| `tasks/` | 可单独交给低成本模型执行的原子任务 |
| `tests/` | 数据结构、泄漏防护和算法单元测试 |
| `outputs/` | 运行日志、预测、指标和任务报告；不提交 Git |

## 5. 实施阶段

1. 冻结旧基线并复算 PHM2010 一个方向。
2. 用户挂载 Hannover，完成真实 HDF5 字段映射。
3. 核验 NASA 原始字段单位，完成跨材料任务清单。
4. 按 `T050 -> T051 -> T052 -> T053 -> T054` 逐个实现局部增强、多尺度编码器、生命周期 TCN 双头、EMA Teacher 和最终集成。
5. 先验证新的 source-only 双头骨干，再逐项加入增强、DARE/RGLA 和 Teacher；不允许一次开启全部模块后归因。
6. 先 1 seed smoke，再 3 seed 筛选，最后 5 seed 正式实验。
7. 只有 VB/RUL 标签可见性审计均为零违规后，结果才可进入论文。
