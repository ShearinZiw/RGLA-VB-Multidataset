# 项目总览：RGLA-VB Multi-dataset

## 1. 研究问题

研究对象是小样本条件下的跨域刀具磨损预测。输入为切削过程多传感器信号和可用工艺元数据，输出为连续磨损量 `VB`。域偏移来自刀具、机床、材料、工况和传感器配置变化。

任务定义不变：

- 输出连续原始 `VB`，不是磨损阶段类别；
- 不以 `VB_norm` 作为训练目标；
- UDA 中目标域 `VB` 在训练和模型选择时完全不可见；
- Prefix-30 属于少标签 SSDA，必须与 UDA 分表报告；
- 不从多个 seed 中挑最好结果，正式结果报告全部预注册 seed 的均值、标准差和配对差值。

## 2. 改进框架

新框架暂命名为 **RGLA-VB：Reliability-Gated Lifecycle-Aware Continuous VB Regression**。它在当前“数据扩增 + DDGAN + 伪标签 + PGRU”的复现路径上增加三个小而可检验的机制：

1. **生成样本可靠性门控**：三头判别器分别输出真实性、域一致性和磨损一致性，用联合置信度降低错误生成样本的权重。
2. **软生命周期建模**：用连续进程变量与源域磨损曲线生成软阶段概率，不把回归任务改成分类。
3. **回归结构对齐**：按软生命周期权重对齐源/目标特征的回归几何结构，避免全局 DANN 把不同磨损水平强行混合。

特征注意力保留为轻量骨干选项，不单独作为主要创新。完整公式见 `docs/01_METHOD_RGLA_VB.md`。

## 3. 证据梯度

| 层级 | 数据 | 作用 | 结论边界 |
|---|---|---|---|
| 开发 | PHM2010 C1/C4/C6 | 与现有复现直接衔接，完成机制消融 | 跨刀具、同数据集 |
| 独立主验证 | Hannover M1/M2/M3 | 6 个真实跨机床有向迁移任务 | 跨机床泛化 |
| 低成本压力测试 | NASA Milling | 8 个同工况、双向跨材料任务 | 小样本跨材料可行性 |

NASA 不能替代 Hannover：它只有 16 个 case、167 个 run，且 21 个 run 缺少 `VB`。它适合做低成本外部方向性验证，不适合单独承担强泛化结论。

## 4. 目录职责

| 路径 | 职责 |
|---|---|
| `legacy/scripts/` | 已有 PHM2010 复现与导出脚本；默认冻结 |
| `src/rgla_vb/data/` | 数据契约、注册表和数据集适配器 |
| `configs/` | 数据路径和预注册实验配置 |
| `docs/` | 方法、数据、实验和验收文档 |
| `tasks/` | 可单独交给低成本模型执行的原子任务 |
| `tests/` | 数据结构、泄漏防护和算法单元测试 |
| `outputs/` | 运行日志、预测、指标和任务报告；不提交 Git |

## 5. 实施阶段

1. 冻结旧基线并复算 PHM2010 一个方向。
2. 用户挂载 Hannover，完成真实 HDF5 字段映射。
3. 核验 NASA 原始字段单位，完成跨材料任务清单。
4. 逐个实现生成可靠性、软生命周期和回归结构对齐。
5. 先 1 seed smoke，再 3 seed 筛选，最后 5 seed 正式实验。
6. 只有通过标签泄漏审计后，结果才可进入论文。
