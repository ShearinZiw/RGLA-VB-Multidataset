# DeepSeek V4 低成本实施指南

## 1. 使用方式

后续不要把“实现整个新框架”一次性交给模型。每次新对话只给：

1. 根目录 `AGENTS.md`；
2. `docs/00_PROJECT_OVERVIEW.md`；
3. `tasks/` 中一个任务文件；
4. 该任务明确列出的源文件。

任务完成、测试通过、人工查看 diff 后，再进入下一任务。旧路线 `T020/T021/T022/T023/T025` 保留用于 B2/RGLA 历史基线，不再直接组成完整主方法。新主线顺序：

```text
T001 -> T010/T011 -> T015 -> T016
     -> T050 生命周期局部增强
     -> T051 多尺度 1D 编码器
     -> T052 生命周期条件 TCN + VB/RUL 双头
     -> T053 不确定性感知 EMA Teacher
     -> T022/T023 回归对齐基线
     -> T054 完整双任务集成
     -> T030 -> T040
```

T050/T051 可在 T016 后独立开发，但 T054 只有在 T050-T053、T022、T023 的 required checks 全部通过后才能开始。

## 2. 固定提示词模板

```text
你在 D:\RGLA-VB-Multidataset 工作。
先读 AGENTS.md、docs/00_PROJECT_OVERVIEW.md 和 tasks/<TASK>.md。
只实施这个任务，只修改 Allowed files，不做额外重构。
任务定义必须保持连续原始 VB 与有审计 EOL 时的 RUL-cut 双回归，禁止使用目标评价 VB/RUL/EOL 训练或选模。
按 Required checks 逐条执行；失败就修复，不要跳过。
完成后列出修改文件、测试输出和仍未验证的假设，并生成任务要求的 report。
```

## 3. 降低模型犯错的约束

- 文件白名单：任务卡没有列出的文件一律不改。
- 一个任务一个机制：不允许同时实现增强、编码器、TCN双头、Teacher和 runner。
- 公式直接给出：模型不得自行替换距离、损失或默认参数。
- 合成数据先行：算法模块先通过可计算期望值的单元测试，再接真实训练。
- 开关回退：每个新模块都必须有默认关闭开关；C0 回退到 source-only 双头，旧 B2 用独立开关复现。
- 显式错误：缺 Hannover 数据、字段或单位时抛错，不允许猜字段和静默填充。
- 输出审计：每次训练都生成标签可见性文件，否则该运行无效。

## 4. 审查清单

人工合并前只回答“是/否”：

| 检查 | 通过条件 |
|---|---|
| 是否只改白名单文件 | `git diff --name-only` 无额外文件 |
| 是否改变任务 | 最终头输出连续原始 VB 与 masked RUL-cut；没有分类头 |
| 是否伪造 RUL | 无可验证 EOL 的样本 `rul_label_mask=0` |
| 是否引入目标标签 | UDA train dataset 不含目标评价 VB/RUL/EOL |
| 是否保留基线 | 新开关关闭时测试完全一致 |
| 是否有数值保护 | 分母有 eps，空阶段明确跳过 |
| 是否可复现 | seed、配置和版本写入输出 |
| 是否真测过 | Required checks 全部有实际输出 |

任一项为“否”都不进入下一任务。

## 5. 成本控制

- 先用 CPU 合成测试验证因果性、双头 shape、RUL mask、EMA stop-gradient，再做 30-cut smoke，禁止直接启动完整五 seed。
- 特征缓存按 dataset/version/hash 复用，避免每个方法重复读取原始信号。
- 三 seed 阶段只保留达到门槛的模块；五 seed 只跑预注册最终矩阵。
- 不让模型自动扩大超参数网格；所有候选值写在实验计划中。
- 日志每个 epoch 一行 JSON，便于较弱模型和脚本机械检查。

## 6. 模型输出不可信时

出现下列情况立即回退该任务 diff：修改 `legacy/` 未获授权、把最后 run 自动当 EOL、把删失样本补成完整 RUL、把缺失 VB 插值为真值、按目标 MAE 选 epoch、选最好 seed、改变单位但不记录换算、删除异常工具而不留审计。不要让同一模型在一个超长对话中反复补丁；重新开一个只做失败测试修复的任务更稳。
